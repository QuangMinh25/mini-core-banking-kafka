# Docs Update Skill

## When To Use

Use for updating AI framework or service documentation in `banking-core-service`.

## Goal

Keep docs concise, project-specific, and tied to actual code or checked-in configuration.

## Required Workflow

1. Identify the code, Gradle file, test, compose file, or config that proves each statement.
2. Replace generic framework wording with shorter service-specific wording where possible.
3. Mark unclear implementation details as `Unknown / needs confirmation`.
4. Update cross-references only where relevant.
5. Report the evidence basis and remaining doc gaps.

## Common Mistakes To Avoid

- Inventing APIs or business flows
- Leaving copy-pasted framework text that no longer helps
- Reading unrelated `.ai` files just to write docs

## Completion Requirements

Include docs updated, evidence basis, verification approach, and remaining gaps.
