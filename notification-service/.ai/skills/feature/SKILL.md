# Feature Skill

## When To Use

Use for adding or expanding behavior in `notification-service`.

## Goal

Implement the feature with clear boundaries, explicit unknowns, and safe verification.

## Required Workflow

1. Confirm requirements and acceptance criteria.
2. Decide whether the work is local to this module or coordinated with `banking-core-service`.
3. Infer conventions from checked-in code instead of from the generic framework.
4. Implement only the required layers.
5. Update docs and tests where the repository supports them.
6. Run safe verification and summarize results honestly.

## Common Mistakes To Avoid

- Inventing missing package conventions
- Mixing refactors into the feature
- Describing undocumented notification flows as implemented

## Completion Requirements

Include feature scope, implemented behavior, changed files, verification result, and follow-up notes.
