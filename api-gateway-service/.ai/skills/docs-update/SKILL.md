# Docs Update Skill

## When To Use

Use for updating repository or project documentation.

## Goal

Keep documentation aligned with actual code, architecture, and approved behavior.

## Required Workflow

1. Identify the source of truth in code, config, or approved requirements.
2. Review the existing docs that are affected.
3. Update only what is supported by evidence.
4. Remove ambiguity where the code is clear; document uncertainty as `Unknown / needs confirmation` where it is not.
5. Report which sources were used and any remaining documentation gaps.

## Checklist

- Documentation tied to real behavior
- No invented APIs or flows
- Examples remain safe and generic
- Cross-references updated where needed

## Common Mistakes To Avoid

- Writing speculative behavior as fact
- Leaving outdated examples unchanged
- Skipping impacted docs outside the immediate file

## Completion Requirements

The result must include docs updated, evidence basis, verification approach, and any areas still needing project-specific detail.
