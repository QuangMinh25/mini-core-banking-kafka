# Issue Analysis Skill

## When To Use

Use when the task is investigation only and no code should be modified.

## Goal

Understand the issue, identify related files and probable causes, and report risks without changing code.

## Required Workflow

1. Read repository guidance and project context.
2. Inspect the relevant modules, configs, queries, logs, or docs.
3. Trace the likely execution path and evidence.
4. Identify probable root causes, impacted areas, and missing information.
5. Report findings and risks without applying changes.

## Checklist

- No files modified
- Related files identified
- Evidence cited from the repository
- Risks and uncertainty made explicit

## Common Mistakes To Avoid

- Making speculative conclusions without evidence
- Quietly changing files during analysis
- Presenting one hypothesis as confirmed fact

## Completion Requirements

The result must include likely cause, related files, confidence level, open questions, and recommended next step.
